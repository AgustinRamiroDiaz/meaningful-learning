# Implementation Plan: AI-Powered Learning Knowledge Graph

**Branch**: `001-ai-learning-knowledge-graph` | **Date**: 2026-04-16 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-ai-learning-knowledge-graph/spec.md`

## Summary

Build a fully client-side Next.js 16 (App Router) application that lets users
create Learning Courses, prompt an AI to generate a concept knowledge graph
(DAG) for any topic, and interactively track their learning progress across
three node states: **known**, **edge of knowledge**, and **future**. The graph
is visualised using React Flow v12 with a dagre auto-layout engine. All data
persists in localStorage. The AI is called directly from the browser using the
`openai` npm package against an OpenAI-compatible endpoint — defaulting to local
Ollama, with the base URL, model, and optional API key configurable via settings.

## Technical Context

**Language/Version**: TypeScript 5 / Node 20 (Next.js 16.2.4, React 19.2.4)
**Primary Dependencies**:
- `next` 16.2.4 (App Router, client-side rendering)
- `react` / `react-dom` 19.2.4
- `@xyflow/react` ^12 — interactive DAG graph canvas
- `@dagrejs/dagre` — automatic DAG layout computation
- `openai` — OpenAI-compatible client (configurable base URL; supports Ollama, OpenAI, OpenRouter)
- `shadcn/ui` (CLI-generated components) + `tailwindcss` v4 — design system
- `husky` + `lint-staged` + `prettier` — pre-commit quality gates (constitution)

**Storage**: Browser localStorage (JSON serialisation; no IndexedDB for PoC)
**Testing**: TypeScript type-check (`tsc --noEmit`) + ESLint; no test framework for PoC
**Target Platform**: Modern evergreen browsers (Chrome/Firefox/Safari/Edge latest 2 versions)
**Project Type**: Client-side web application (static export compatible)
**Performance Goals**: Initial load interactive <3s; graph state re-evaluation <300ms
**Constraints**: No server infrastructure; API key user-supplied and stored in localStorage; graphs 5–50 nodes
